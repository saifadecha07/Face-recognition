# Current Capabilities

## ตอนนี้ระบบทำอะไรได้บ้าง

ระบบที่แก้แล้วสามารถทำงานได้ดังนี้

1. เปิดรับภาพจากกล้องจริง
2. ตรวจจับใบหน้าในภาพแบบต่อเนื่อง
3. เทียบใบหน้ากับ dataset ที่เก็บไว้เพื่อระบุว่าเป็นใคร
4. ถ้าจำไม่ได้ จะติดป้ายเป็น `Unknown`
5. ติดตามคนเดิมข้ามหลายเฟรมด้วย `track_id`
6. แยกได้ว่าคนนั้น:
   - เพิ่งเข้ามาในกล้อง
   - ยังอยู่ในกล้อง
   - ออกจากกล้องไปแล้ว
7. วัดการขยับของตำแหน่งในภาพเพื่อดูว่ากำลังเคลื่อนไหวหรือไม่
8. บันทึกสถานะทั้งหมดลง PostgreSQL
9. เปิด API เพื่อให้ระบบอื่นหรือ dashboard มาอ่านข้อมูลได้
10. รันบน server ของตัวเองผ่าน Docker ได้

## Event ที่ระบบรู้จัก

- `entered`
  - คนนี้เพิ่งถูกพบในกล้องครั้งแรกของ session นี้
- `visible`
  - คนนี้ยังอยู่ในกล้อง และข้อมูลถูกอัปเดตต่อเนื่อง
- `exited`
  - คนนี้หายจากภาพเกินจำนวนเฟรมที่กำหนด ระบบจึงถือว่าออกจากกล้องแล้ว

## ข้อมูลที่บันทึกต่อคนต่อหนึ่ง session

- `track_id`
- `camera_name`
- `label`
- `status`
- `first_seen_at`
- `last_seen_at`
- `left_at`
- `confidence`
- `movement_score`
- พิกัดกล่องล่าสุดของใบหน้า

## API ที่เรียกใช้ได้

- `GET /health`
- `GET /persons`
- `GET /sightings/active`
- `GET /sightings/recent`
- `POST /camera/start`
- `POST /camera/stop`

## งานที่คุณเอาไปต่อได้ทันที

- ทำ dashboard ดูว่าใครอยู่หน้ากล้องตอนนี้
- ทำระบบ attendance หรือ access log
- ทำระบบเชื่อมกับกล้อง RTSP/IP camera
- ทำระบบแจ้งเตือนใหม่ผ่าน LINE, email, webhook หรือ mobile app
- ทำระบบหลายกล้องโดยแยก `camera_name`
- ทำระบบสิทธิ์เข้าออกพื้นที่
